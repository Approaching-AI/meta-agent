## 讨论：为项目编写方法论 doc

和人类讨论了这套 agent 工作方法论的核心要点，准备写一份 doc 来阐明整体工作方式。以下是讨论中确认的内容：

### 核心要素

- **Daily notes** — agent 的工作日志，append-only，积累实践经验
- **Doc** — 知识库，来源多样（AI 总结、人类编写、外部文档），组织在 `doc/` 下
- **SOP** — 从 daily notes 中提炼的操作流程，存放在 `doc/sop/`

### 工作流程

- Agent 由 prompt 驱动开始工作，按需查阅 daily notes 和 doc
- Agent 自行判断需要读什么，prompt 提供指导
- 信息沉淀路径：daily notes → SOP
- Doc 独立积累，不限来源

### 分工

- Daily notes 由 agent 维护
- SOP 创建由 human 决定，agent 可建议
- Doc 人和 agent 都可以写

### 设计原则

- 框架无关、模型无关，只定义方法论，不依赖特定 agent 实现
- Memory 不纳入体系，由各 agent 工具自行管理

## 完成：编写 doc/methodology.md

基于上述讨论，完成了 `doc/methodology.md` 的编写。文档包含以下内容：

10. 开放问题：多 agent 协作（合并与分裂）

另外讨论中确认的额外要点：
- Agent Native 的概念：不仅是从 Day 0 用 agent，也是用 agent 作为协作媒介
- 上下文传递困难的论证逻辑：agent 能力不是瓶颈 → 上下文才是 → 人传递上下文效率低 → 所以让 agent 自己积累
- "均摊成本"：文档沉淀分摊到每次 session，而非事后补齐
